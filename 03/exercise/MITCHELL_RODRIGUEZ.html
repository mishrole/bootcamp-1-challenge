  1. Que es un sistema de control de versiones?

    Es un sistema de registro de cambios que permite almacenar, controlar, comparar y revertir cambios en un archivo o conjunto de archivos a través del tiempo, haciendo posible la recuperación de versiones específicas, y permitiendo el trabajo de N colaboradores sobre una misma fuente de código de manera efectiva.

  2. Cual es la diferencia entre git y github?

    Git es un sistema de control de versiones, mientras que Github es un sistema de gestión de código abierto que funciona sobre Git, potenciando sus funcionalidades y añadiendo propias como el manejo de tickets o el manejo de proyectos. Github es, además, una red social que permite a desarrolladores exponer sus proyectos a la comunidad y recibir aportes.
